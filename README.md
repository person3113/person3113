# 안녕하세요, 정성오입니다.

**"AI를 통해 빠르게 만들고, 뜯어보며 내 것으로 만듭니다."**

현재 대학생이며 자바 스프링(Java Spring) 백엔드 개발을 공부하고 있습니다.
처음부터 완벽한 코드를 짜기보다, **AI를 활용해 일단 동작하는 결과물을 빠르게 만들고, 이후 코드 리뷰와 리팩토링을 통해 원리를 깊게 파고드는 방식**으로 개발합니다.

이론만 아는 개발자가 아니라, 부딪히며 문제를 해결하는 개발자가 되기 위해 기록하고 있습니다.

## 저의 개발 철학

**1. 일단 돌아가게 만듭니다 (Fast Cycle)**
* 완벽한 설계에 집착하기보다 핵심 기능부터 빠르게 구현합니다.
* AI 도구(Gemini CLI, OpenCode 등)를 적극 활용하여 개발 생산성을 높입니다.

**2. 하지만 반드시 되돌아봅니다 (Refactoring & Learning)**
* AI가 짜준 코드는 제 것이 아닙니다. 반드시 코드를 주기적으로 점검하고 이해하는 과정을 거칩니다.
* 구현이 끝난 후, 유지보수가 좋지 않은 코드나 비효율적인 부분은 시간을 들여 제 손으로 다시 고칩니다.

---

## 기술 스택

* **Backend:** Java, Spring/SpringBoot, Spring Data JPA, QueryDSL
* **Frontend:** Thymeleaf, HTML/CSS/JS (Basic)
* **Database:** RDBMS(Basic SQL)
* **Tools:** Git/Github, Notion, AI Tools(Gemini CLI, OpenCode, Github Copilot)

## 지금까지의 개발 기록

### 1. 맨땅에 헤딩하며 배우기: 이론과 실습 사이
> *"강의만 보니까 머리에 남지 않아서, 코드를 직접 고쳐보기 시작했습니다."*

* **김영한님 스프링 강의 & JPA Shop 변형**
    * 강의를 듣는 것에 그치지 않고, `jpashop` 예제 코드를 제 나름대로 수정해보며 학습했습니다.
    * 단순 조회 화면의 성능 문제(N+1 등)를 해결하기 위해 DB 설계를 다시 해보고, 화면에 필요한 쿼리를 최적화하는 고민을 했습니다.

### 2. 나만의 개발 방식 찾기: SBB 게시판 7일 챌린지
> *"작은 기능부터 차근차근, 계획과 구현을 분리해서 진행했습니다."*

* **SBB(스프링 부트 게시판) 프로젝트**
    * 노베이스 상태에서 7일 안에 게시판을 만드는 것을 목표로 진행했습니다.
    * **배운 점:** 처음부터 거창한 기획을 하기보다, 작은 기능(MVP)부터 단계적으로 구현하는 것이 효율적임을 배웠습니다. '계획(가이드라인 작성)'과 '실행(코드 구현)' 단계를 분리하여 AI를 활용했을 때 시행착오가 가장 적다는 것을 깨달았습니다.

### 3. 실전 적용: 팀 프로젝트 'RecipeMate' (25년 2학기)
> *"AI를 활용해 팀 개발의 리드와 생산성을 모두 잡았습니다."*

* **교내 웹 프로그래밍 프로젝트 (Java & Spring)**
    * **역할:** 5인 팀 프로젝트였으나, 팀원 간 실력 차이로 인해 주도적으로 개발을 이끌었습니다. (기여도: 메인 개발 및 구조 설계)
    * **AI 활용의 진화:** 이전 프로젝트(국회.gg) 때 겪었던 시행착오를 바탕으로, 이번에는 `OpenCode`와 `Gemini CLI`를 활용했습니다. 단순 코드 생성을 넘어 디버깅과 구조 분석에 AI를 적극적으로 활용해, 짧은 기간과 늦은 기획 확정에도 불구하고 완성도 있는 결과물을 냈습니다.
    * **계획:** 학기 중에 마감 기한 때문에 타협했던 코드들을 이번 겨울 방학 때 다시 뜯어보고 개선(리팩토링)할 예정입니다.
